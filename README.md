CRM Sales Opportunities Analysis (SQL Server)
Project Overview
This project focuses on analyzing a CRM sales pipeline dataset to extract actionable business insights. Using SQL Server Management Studio (SSMS), I developed a series of advanced queries to track sales performance, analyze agent productivity, and identify revenue trends. The project demonstrates proficiency in complex SQL techniques including CTEs, Window Functions, and Data Aggregation.

Dataset Description
The analysis is based on five interconnected tables representing a company's sales ecosystem:

accounts: Details of client companies including sector, revenue, and headquarters.

products: Information on product offerings, series, and retail prices.

sales_teams: Mapping of sales agents to their managers and regional offices.

sales_pipeline: The core transactional table containing deal stages (Prospecting, Engaging, Won, Lost) and deal values.

data_dictionary: Reference for all fields and table definitions.

Key Technical Skills Demonstrated
Advanced Aggregations: Summarizing sales performance across regions and managers.

Window Functions: Utilizing RANK(), DENSE_RANK(), and SUM() OVER() for performance leaderboard and cumulative revenue tracking.

Common Table Expressions (CTEs): Breaking down complex logic into readable, modular blocks.

Time-Series Analysis: Calculating Year-over-Year (YoY) growth and Month-over-Month (MoM) trends using date manipulation.

Data Cleaning & Transformation: Handling nulls and formatting data for SSMS-compatible reporting.

Analysis Framework
The SQL scripts included in this repository answer critical business questions such as:

Revenue Performance: What is the total revenue generated by "Won" deals per quarter?

Sales Velocity: How long does it take, on average, to move a deal from the "Engaged" stage to "Won"?

Agent Productivity: Ranking agents by their conversion rates and total close values.

Pipeline Health: Identifying the current distribution of deals across various stages to forecast future revenue.

Market Penetration: Analyzing which sectors (e.g., Technology, Medical) contribute most to the bottom line.

How to Use This Repository
Database Setup: Create a new database in SSMS and import the provided .csv files using the Import Flat File wizard.

Queries: Navigate to the /Scripts folder to find categorized SQL tasks ranging from basic selections to advanced analytical queries.

Documentation: Refer to the data_dictionary.csv for detailed column descriptions and relationships.
